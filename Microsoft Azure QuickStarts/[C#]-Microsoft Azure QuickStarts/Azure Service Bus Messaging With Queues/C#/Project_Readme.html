Azure Service Bus: Messaging With Queues

Microsoft Azure QuickStarts

Simple demonstration of how to use Azure Service Bus Queues for messaging. This approach can be applied to de-couple backend and frontend roles and can also be used to load balance and load level processing for variable load scenarios.

If you don't have a Microsoft Azure subscription you can get a FREE trial account here.

This QuickStart demonstrates the following scenarios:

  1. Creating a Service Bus Queue
  2. Sending messages to a Queue
  3. Receiving messages from a Queue
  4. Handling transient errors.

Running this sample

  1. Open the Azure Management Portal to create a service bus namespace and retrieve the connection string details (see the documentation for details)
  2. Open app.config and update [your namespace] with your service bus namespace and [your access key] with the access key for the corresponding namespace
  3. Run the project
